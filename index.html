CVE-2021-44228分析

发表于 2025-01-19 分类于 Security ， Vulnerability Analysis

CVE-2021-44228

这是漏洞的触发代码：

  • 创建了一个Logger实例
  • 构造了一个包含JNDI注入payload的字符串
  • 通过logger.error()输出日志，其中包含恶意payload

error方法：

  • 接收日志消息模板和参数
  • 调用logIfEnabled方法处理日志

logIfEnabled方法：

  • 检查是否启用了对应级别的日志
  • 如果启用则调用logMessage处理

logMessage方法：

  • 通过messageFactory创建新的消息对象
  • 调用logMessageSafely继续处理

logMessageSafely方法：

  • 调用logMessageTrackRecursion处理日志
  • 最后释放消息对象资源

logMessageTrackRecursion方法：

  • 增加递归深度计数
  • 调用tryLogMessage实际处理日志
  • 最后减少递归深度计数

tryLogMessage方法：

  • 尝试调用log方法处理消息
  • 捕获异常并进行处理
  • 使用@PerformanceSensitive注解表明这是性能敏感的代码

这是一个编译后的中间方法，继续向下传递日志事件。

这个log方法：

  • 获取当前活动的LoggerConfig配置
  • 调用配置的log方法处理日志事件
  • 最后执行可靠性策略的后处理

这个方法是漏洞触发的关键环节：

  1. 处理日志配置的属性
  2. 如果属性需要查找(lookup)，就会创建LogEvent并执行字符串替换
  3. 字符串替换过程中会解析${jndi:...}这样的表达式
  4. 当遇到jndi前缀时会触发JNDI查找，从而执行恶意代码
  5. 最后创建完整的日志事件并进行处理
